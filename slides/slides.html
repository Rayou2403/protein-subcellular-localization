Protein Subcellular Localization

A Deep Learning Approach using ESM-C and ProstT5 3Di Embeddings

Rayane BELKALEM & Moh-Ouali AMROUNI

Project Implementation & Architecture Design

Project Objectives

Re-Implementation

Accurate reproduction of the reference pipeline using modern pLM embeddings. Emphasis on data rigor and biological validity.

Novel Architecture

Design of a custom Deep Learning model. Combining ESM-C (Sequence) and ProstT5 3Di (Structure) for enhanced prediction.

Input Embeddings

1. ESM-C (Evolutionary Scale)

A powerful sequence-based model. We utilize the pooling attention mechanism to capture global sequence properties. Due to its size, we implement CPU offloading.

2. ProstT5 3Di (Structural)

Provides structure-aware embeddings. We use aa2fold translation followed by mean pooling to encapsulate 3D structural information into the feature space.

Proposed Architecture

Feature Fusion Strategy

We propose a Concatenation + MLP architecture:

  • Input: Dual embeddings (ESM-C + ProstT5) are concatenated to form a unified feature vector.
  • Hidden Layers: A custom Multi-Layer Perceptron (MLP) with Batch Normalization and ReLU activation to learn non-linear interactions.
  • Regularization: Dropout layers (p=0.3) are injected to prevent overfitting on the training set.
  • Output: A final Softmax layer maps the features to the target subcellular localization classes.

Data Integrity & Robustness

No Data Leakage

Strict separation of Train, Validation, and Test sets. Homology reduction ensures no similar sequences appear across splits.

Class Imbalance

Implementation of Weighted Cross-Entropy Loss. Weights are inversely proportional to class frequencies to penalize majority bias.

Evaluation Metrics

Primary metric: F1-Macro score to account for imbalance, alongside overall Accuracy for general performance.

Technical Challenges & Solutions

Resource Management

The ESM-C model is memory-intensive. To avoid OOM (Out Of Memory) errors on standard GPUs:

  • CPU Offloading: ESM-C is forced to run in CPU mode for embedding extraction.
  • Batch Size 1: Sequential processing ensures stability.
  • Subsetting: Implemented --subset_frac 0.1 flag for rapid debugging.

Dependencies

Patched the tokenizer to resolve compatibility issues between ESM-C and Transformers library versions (4.35.2).
